🎯 Ejemplos de Formularios en Angular

Demostración de Template-driven Forms vs Reactive Forms

📝 1. Template-Driven Form (Formulario Dirigido por Plantilla)

Usa ngModel para enlace bidireccional. Validaciones en HTML. Ideal para formularios simples.

ng-valid ng-invalid ng-touched ng-dirty
@if (nombreInput.invalid && (nombreInput.dirty || nombreInput.touched || templateSubmitted)) {
@if (nombreInput.errors?.['required']) { ❌ El nombre es requerido } @if (nombreInput.errors?.['minlength']) { ❌ Debe tener al menos 3 caracteres }
}
ng-valid ng-invalid ng-touched
@if (emailInput.invalid && (emailInput.dirty || emailInput.touched || templateSubmitted)) {
@if (emailInput.errors?.['required']) { ❌ El email es requerido } @if (emailInput.errors?.['email']) { ❌ Debe ser un email válido }
}
ng-valid ng-invalid
@if (edadInput.invalid && (edadInput.dirty || edadInput.touched || templateSubmitted)) {
@if (edadInput.errors?.['required']) { ❌ La edad es requerida } @if (edadInput.errors?.['min']) { ❌ Debe ser mayor de 18 años } @if (edadInput.errors?.['max']) { ❌ Edad máxima: 100 años }
}
Estado del Formulario: @if (templateForm.valid) { ✅ Válido } @else { ❌ Inválido }

📊 Vista de Datos (ngModel binding):

{{ templateUser | json }}

⚡ 2. Reactive Form (Formulario Reactivo)

Usa FormGroup y FormControl. Validaciones en TypeScript. Ideal para formularios complejos y profesionales.

ng-valid ng-invalid ng-touched ng-dirty
@if (hasError('nombre', 'required')) {
❌ El nombre es requerido
} @if (hasError('nombre', 'minlength')) {
❌ Debe tener al menos 3 caracteres
}
ng-valid ng-invalid
@if (hasError('email', 'required')) {
❌ El email es requerido
} @if (hasError('email', 'email')) {
❌ Debe ser un email válido
}
@if (hasError('edad', 'required')) {
❌ La edad es requerida
} @if (hasError('edad', 'min')) {
❌ Debe ser mayor de 18 años
} @if (hasError('edad', 'max')) {
❌ Edad máxima: 100 años
}
@if (hasError('password', 'required')) {
❌ La contraseña es requerida
} @if (hasError('password', 'minlength')) {
❌ Debe tener al menos 6 caracteres
}
@if (hasError('confirmPassword', 'required')) {
❌ Debe confirmar la contraseña
}

Touched: {{ formTouched }} Dirty: {{ formDirty }}

📊 Vista de Datos (FormGroup value):

{{ reactiveForm.value | json }}

Usa FormControl de forma independiente, sin necesidad de FormGroup. Ideal para búsquedas, filtros, campos individuales.

🏷️ 4. Template Reference Variables (#)

Usa #variableName para referenciar elementos DOM en la plantilla. Útil para acceder a propiedades y métodos de elementos HTML.

📊 Comparación: Template-driven vs Reactive

📝 Template-driven

  • ✅ Más simple y rápido
  • ✅ Ideal para formularios pequeños
  • ✅ Menos código TypeScript
  • ❌ Validaciones en HTML
  • ❌ Difícil de testear

⚡ Reactive

  • ✅ Más control y flexibilidad
  • ✅ Ideal para formularios complejos
  • ✅ Validaciones dinámicas
  • ✅ Fácil de testear
  • ❌ Más código TypeScript
